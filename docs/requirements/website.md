# Website Requirements: Dissertation Showcase

## Context

A modern, static website to showcase a dissertation, built with SvelteKit and
hosted on GitHub Pages. The site will be simple, easy to maintain, and primarily
serve as a landing page with a few additional sections.

## Use Case

A visitor lands on the website to learn about the dissertation, view publication
details, and find contact information. The site is intended for academic and
professional audiences.

## Tasks

Execute the following tasks:

1. Set up a SvelteKit project configured for static site generation and GitHub
    Pages deployment.
1. Implement a main landing page with dissertation title and subtitle
1. Add additional sections:
    1. About
    1. Publication
    1. Contact (display info only)
1. Navigation: Add a navigator bar at the top of the website when clicked it jumps to each section
1. Apply a modern design using the color palette:
    1. RGB(219, 171, 194)
    1. RGB(166, 216, 209)
1. Make the font configurable (use a placeholder font initially)

## Non-functional requirements

During all tasks upheld the following non-functional requirements

1. Language: Ensure the site is single-language (Dutch)
1. Use code-based content management (no admin interface)
1. Ensure cross-device and cross-browser compatibility
1. Implement simple SEO (title, meta description)

## Acceptance Criteria

The website is complete if

1. The landing page displays the dissertation title, navigation, and a
    placeholder image
1. Navigation links work and scroll to/about each section
1. The site builds and deploys successfully to GitHub Pages
1. About, Publication, and Contact sections/pages exist with placeholder content
1. Basic SEO tags are present (title, meta description)
